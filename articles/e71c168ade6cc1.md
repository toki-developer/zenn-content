## 必要な知識について
* Reactが分かれば基本は困らずに開発できる
* Widgetなど、よりOSに近い機能を使う場合はネイティブの知識が求められる
* 実機でビルドする時とかに少し詰まる


## 開発時のイメージ

* npm run iosで動かす
* ホットリロードで開発便利
* npm run ios:deviceで使う端末も切り替えられる
* swift側のコードを修正した場合も、npm run iosを実行すれば反映できる。jsサーバーの方は止める必要はない。

## リリースまわり

### 基本フロー

* TestFlightにあげる
* App Store Connectに入力する
  * TestFlightからリリースするビルドを選ぶ
  * 基本的には上から順に埋めていけばOK
    * ローカル完結アプリ & 課金なしなので、難しい記入箇所がなかった。別の機会に学びたい。
* 申請に出す

### EASありでのリリース
* eas-buildとeas-submitだけで簡単にTestFlightにあげれる
* 自動でProfileとか作ってくれる

### EASなしでのリリースについて
* EASなしでも簡単にリリースできる
  - ローカル環境でのビルドとリリースもドキュメントが整備されていて簡単
  - 参考: [Local app production](https://docs.expo.dev/guides/local-app-production/#ios)
* EASで作ったProfileとか使えば良い。うまくいかなければ、自分でProfile作ってやる
